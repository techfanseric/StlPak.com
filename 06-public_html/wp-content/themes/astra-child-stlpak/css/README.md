# CSS模块化重构说明

## 重构概述

原始的 `style.css` 文件有9636行，大小为212KB。为了提高网站性能和可维护性，已将其拆分为10个模块化的CSS文件。

## 文件结构

### 基础文件（所有页面都会加载）

1. **base.css** (24K)
   - 重置样式
   - 全局样式
   - 工具类
   - 响应式媒体查询

2. **components.css** (8K)
   - 按钮样式
   - 表单组件
   - 弹窗组件
   - 通用交互元素

3. **header.css** (12K)
   - 头部导航
   - 顶部栏
   - Mega菜单
   - 响应式导航

4. **footer.css** (4K)
   - 页脚样式
   - 联系信息
   - 社交媒体链接

### 页面特定文件（按需加载）

5. **home.css** (88K)
   - 所有首页变体（home1-16）
   - 主页特定组件
   - 轮播图
   - 产品展示

6. **about.css** (16K)
   - 关于我们页面
   - 公司历史
   - 团队介绍

7. **category.css** (28K)
   - 产品分类页面
   - 筛选器
   - 产品网格

8. **detail.css** (16K)
   - 产品详情页
   - 图片轮播
   - 产品规格

9. **contact.css** (8K)
   - 联系页面
   - 联系表单
   - 地图样式

10. **landing.css** (20K)
    - 着陆页
    - 博客页面
    - FAQ页面
    - 特殊功能页面

## 性能优化

### 加载策略

- **基础样式**：所有页面加载，确保基本功能正常
- **页面特定样式**：根据页面模板动态加载
- **总计**：首页只需要加载约60K的CSS（对比原来的212K）

### 优势

1. **减少初始加载时间**：只加载必要的CSS
2. **更好的缓存利用**：基础样式被所有页面共享
3. **提高维护性**：每个模块职责单一
4. **便于团队协作**：不同开发者可以并行编辑不同模块

## 使用方法

CSS文件已通过 `functions.php` 自动管理。系统会根据当前页面类型自动加载相应的CSS文件。

### 手动加载特定CSS

如需手动加载，可以使用：

```php
wp_enqueue_style( 'stlpak-home-css', get_stylesheet_directory_uri() . '/css/home.css' );
```

## 备份文件

- `style-original.css` - 原始CSS备份
- `functions-original.php` - 原始functions.php备份

## 注意事项

1. 修改CSS时，请编辑 `/css/` 目录下的对应文件
2. 新增页面类型时，需要在 `functions.php` 中添加相应的加载逻辑
3. 确保新增的CSS文件包含完整的主题头信息
